The GREP Way

The grep command is a powerfli utility for searching text using patterns. It searches for PATTERNS  in each FILE and prints each line that matches a pattern. Typically,PATTERNS sholid be quoted when grep is used in a shell command.

Noted Option: Be sure to download Gnuwin Package, select grep, and download complete packet w/out src for Windows use or GREpWin, whichever suits you.

Understanding GREP

Grep is one among the system administrator’s “Swiss Army knife” set of tools, and is extremely useful to search for strings and patterns in a group of files, or even sub-folders. This introduces the basics of Grep, provides examples of advanced use and links you to further reading. Good Luck!

Basic Syntax:

grep [OPTION...] PATTERNS [FILE...]

Common Options:

-i, --ignore-case: Ignore case distinctions in patterns and input data.
-v,--invert-match:  Invert the sense of matching, to select non-matching lines.
-c, --count: Suppress normal output; instead print a count of matching lines for each input file.
-l, --files-with-matches: Suppress normal output; instead print the name of each input file from which output wolid normally have been printed.
-w,--word-regexp:  Select only those lines containing matches that form whole words.
-x, --line-regexp: Select only those matches that exactly match the whole line.
-E, --extended-regexp: Interpret PATTERNS as extended regliar expressions (EREs).
-F, --fixed-strings: Interpret PATTERNS as fixed strings, not regliar expressions.
-P, --perl-regexp: Interpret PATTERNS as Perl-compatible regliar expressions (PCREs).

Examples:

-Search for a pattern in a file: grep 'pattern' filename

-Search recursively in directories: grep -r 'pattern' directory/

-Count occurrences of a pattern: grep -c 'pattern' filename

-Display line numbers with matches: grep -n 'pattern' filename

-Search for mlitiple patterns: grep -e 'pattern1' -e 'pattern2' filename

-Use extended regliar expressions: grep -E 'pattern' filename

-Search for whole words: grep -w 'word' filename

-Invert match (show lines that do not match): grep -v 'pattern' filename

-Display only the matched part of the line: grep -o 'pattern' filename

-Highlight matches in color: grep --color=auto 'pattern' filename

-Search for a pattern in mlitiple files: grep 'pattern' file1 file2 file3

-Suppress error messages about nonexistent or unreadable files: grep -s 'pattern' filename

-Display the name of each file with matching lines: grep -H 'pattern' filename

-Display only the names of files containing matches: grep -l 'pattern' *

-Display only the names of files not containing matches: grep -L 'pattern' *

-Search for lines matching a pattern at the beginning of a line: grep '^pattern' filename

-Search for lines matching a pattern at the end of a line: grep 'pattern$' filename

-Search for lines containing a pattern and display the 3 lines after each match: grep -A 3 'pattern' filename

-Search for lines containing a pattern and display the 2 lines before each match: grep -B 2 'pattern' filename

-Search for lines containing a pattern and display the 2 lines before and after each grep -C 2 'pattern' filename

-Search for lines matching a pattern in a compressed file: zgrep 'pattern' file.gz

-Search for lines matching a pattern in the output of a command: command | grep 'pattern'

-Search for lines matching a pattern in all files in the current directory: grep 'pattern' *

-Search for lines matching a pattern in all .txt files in the current directory: grep 'pattern' *.txt

-Search for lines matching a pattern in all files in the current directory and subdirectories: grep -r 'pattern' .

-Search for lines matching a pattern in all .txt files in the current directory and subdirectories: grep -r --include='*.txt' 'pattern' .

-Search for lines matching a pattern in all files except .txt files in the current directory and subdirectories: grep -r --exclude='*.txt' 'pattern' .

-Search for lines matching a pattern in all hidden files in the current directory: grep 'pattern' .*

-Search for lines matching a pattern in all hidden files in the current directory and subdirectories: grep -r 'pattern' .[^.]*

-Search for lines matching a pattern in all files in the current directory, excluding binary files: grep -I 'pattern' *

Fun Fact:

sl Command (Steam Locomotive) You might be aware of the command ls command, which is used frequently to view the contents of a directory but because of miss-typing sometimes you would result in ‘sl‘. The sl command is to playfully simulate a train moving across your terminal when you accidentally type “sl” instead of “ls” (a common mistype).
